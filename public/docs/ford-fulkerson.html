Ford-Fulkerson Method

The Ford-Fulkerson algorithm computes the maximum flow in a flow network. It relies on finding "augmenting paths" in a residual graph.

How it Works

  1. Start with 0 flow on all edges.
  2. Create a Residual Graph. This graph shows the *remaining* capacity on edges, and also allows "undoing" flow by adding backward edges.
  3. Find an augmenting path from the source to the sink in the residual graph using DFS or BFS. An augmenting path is just a simple path where every edge has a remaining capacity > 0.
  4. Find the edge with the minimum capacity along this path (this is the bottleneck).
  5. Add this bottleneck value to the total maximum flow.
  6. Update the residual graph: subtract the bottleneck from the forward edges, and add it to the backward edges.
  7. Repeat steps 3-6 until no more augmenting paths can be found!

The Problem with Ford-Fulkerson

Ford-Fulkerson is technically a "method", not a fully defined algorithm, because it does not specify *how* you should find the augmenting path in step 3. If you use Depth-First Search (DFS) to find paths, and the capacities are very large or irrational numbers, the algorithm can get stuck in a loop of tiny increments and take an incredibly long time to finish.

// Ford-Fulkerson (Conceptual Logic)
function fordFulkerson(graph, source, sink) {
  let maxFlow = 0;
  let residualGraph = buildResidualGraph(graph); // Creates backward edges
  
  while (true) {
    // findPath can be DFS
    let path = findPath(residualGraph, source, sink);
    if (!path) break; // No more paths, we are done!
    
    let bottleneck = findBottleneck(path);
    maxFlow += bottleneck;
    
    updateResidualGraph(residualGraph, path, bottleneck);
  }
  
  return maxFlow;
}
