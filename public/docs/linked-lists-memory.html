Linked Lists in Memory

Unlike Arrays, Linked List nodes are scattered throughout the computer's memory. They don't need contiguous memory blocks because each node holds a specific memory address (pointer) pointing to the exact location of the next node.

Memory Layout

When you declare an array, the operating system must find a single, unbroken block of memory large enough to hold the entire array. If you need to store 10,000 integers, you need space for 10,000 integers right next to each other.

A Linked List, however, allocates memory dynamically as nodes are created. Node 1 might be at memory address `0x00A1`, while Node 2 might be at `0x0F4B`. The linked list works perfectly fine because Node 1 contains the data AND the address `0x0F4B`.

Figure: Scattered nodes pointing to each other using explicit memory addresses.

Advantages in Memory Allocation

  • No Wasted Space: You only use the memory you need. Arrays often allocate extra space "just in case" to avoid resizing.
  • Easy Insertion/Deletion: You don't have to shift 1,000 elements in memory to insert a new node at the beginning; you just change one pointer.

Disadvantages

  • Pointer Overhead: Every piece of data requires extra memory to store the pointer(s). In a 64-bit system, every pointer is 8 bytes.
  • No Random Access: Because memory is scattered, the CPU cannot instantly calculate the address of the 5th element. It must traverse from the head, node by node.
  • Cache Misses: Modern CPUs load contiguous chunks of memory into their ultra-fast cache. Since linked lists are scattered, traversing them often causes cache misses, making them slower than arrays for sequential reads.
