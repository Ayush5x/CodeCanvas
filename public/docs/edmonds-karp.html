Edmonds-Karp Algorithm

The Edmonds-Karp algorithm is a specific implementation of the Ford-Fulkerson method. The key difference is that it strictly uses Breadth-First Search (BFS) to find the augmenting paths in the residual graph.

Why Use BFS?

By using BFS, Edmonds-Karp guarantees that the augmenting path it finds is the shortest path from the source to the sink (in terms of the number of edges, not capacity). This seemingly small change completely fixes the infinite loop / slow performance issue of Ford-Fulkerson on graphs with large or irrational capacities.

Time Complexity

Because it uses BFS, the time complexity is bounded mathematically to `O(V * E^2)` where V is the number of vertices and E is the number of edges. This makes it a polynomial time algorithm, meaning it will always finish in a reasonable amount of time regardless of how large the edge capacities are.

// Edmonds-Karp (Conceptual Logic)
function edmondsKarp(graph, source, sink) {
  let maxFlow = 0;
  let residualGraph = buildResidualGraph(graph);
  
  while (true) {
    // IMPORTANT: findPath MUST use Breadth-First Search!
    let path = bfsFindPath(residualGraph, source, sink);
    if (!path) break; 
    
    let bottleneck = findBottleneck(path);
    maxFlow += bottleneck;
    
    updateResidualGraph(residualGraph, path, bottleneck);
  }
  
  return maxFlow;
}
