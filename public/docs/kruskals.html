Kruskal's Algorithm

Kruskal's algorithm is another greedy algorithm that finds a Minimum Spanning Tree for a connected, undirected graph. Instead of building the tree from a single starting vertex, it sorts all the edges and adds them one by one.

How it Works

Kruskal's algorithm heavily relies on a data structure called Disjoint-Set (Union-Find) to detect cycles efficiently.

  1. Sort all the edges from lowest weight to highest weight.
  2. Take the edge with the lowest weight and add it to the spanning tree.
  3. If adding the edge creates a cycle, reject this edge instead.
  4. Keep adding edges until there are exactly `V - 1` edges in the spanning tree.

Prim's vs Kruskal's

  • Prim's Algorithm works better on dense graphs (many edges) because it grows from a single point and doesn't need to sort all edges.
  • Kruskal's Algorithm works better on sparse graphs (fewer edges) because sorting fewer edges is very fast.
// Kruskal's Algorithm (Conceptual)
function kruskals(vertices, edges) {
  const mst = [];
  const uf = new UnionFind(vertices); // Custom Disjoint Set class
  
  // Sort edges by weight
  edges.sort((a, b) => a.weight - b.weight);
  
  for (let edge of edges) {
    // If they are not in the same set, no cycle will form
    if (uf.find(edge.u) !== uf.find(edge.v)) {
      uf.union(edge.u, edge.v);
      mst.push(edge);
    }
    
    // Stop early if we have V-1 edges
    if (mst.length === vertices - 1) break;
  }
  
  return mst;
}
