Linked Lists Types

There are three main variants of the Linked List data structure: Singly, Doubly, and Circular Linked Lists. Each type offers specific advantages for different algorithms.

1. Singly Linked List

This is the most common and simple type. Navigation is strictly forward-only. Each node points to the next node in the sequence, and the last node points to null.

  • Advantage: Minimal memory overhead (only one pointer per node).
  • Disadvantage: Cannot easily traverse backwards or delete a node if you only have a pointer to that node (you need to traverse from the head to find the previous node).

2. Doubly Linked List

In a Doubly Linked List, each node contains two pointers: one to the next node, and one to the previous node. The first node's previous pointer is null.

  • Advantage: Can traverse the list in both directions. Deleting a given node is O(1) if you have the pointer to it.
  • Disadvantage: Higher memory overhead due to the extra pointer in every node. More complex code to handle insertion/deletion.

Figure: A Doubly Linked List with prev and next pointers.

3. Circular Linked List

In a Circular Linked List, the last node does not point to null. Instead, it points back to the first node (head), forming a continuous circle. This can be implemented as a Singly Circular or Doubly Circular list.

  • Advantage: Useful for applications that need to continually loop through a list of items (like a music playlist or CPU scheduling).
  • Disadvantage: Risk of infinite loops if traversal conditions are not carefully written.
