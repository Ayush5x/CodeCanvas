DSA Simple Algorithm

An algorithm is a step-by-step procedure or formula for solving a problem. Every program you write is essentially an algorithm. Understanding simple algorithms is the foundation of computer science and helps you think logically about problems.

What Makes a Good Algorithm?

A well-designed algorithm should satisfy the following properties:

  • Correctness: The algorithm must produce the correct output for every valid input.
  • Efficiency: It should use minimal time and memory to solve the problem.
  • Clarity: The steps should be unambiguous and easy to understand.
  • Finiteness: It must terminate after a finite number of steps.
  • Generality: It should work for a range of inputs, not just one specific case.

Your First Algorithm: Finding the Largest Number

Let's write a simple algorithm to find the largest number in a list. This is an excellent first example because it teaches us about iterating, comparing, and tracking state.

Algorithm (Plain English):

  1. Start by assuming the first number in the list is the largest.
  2. Go through each remaining number in the list.
  3. If the current number is greater than your assumed largest, update your assumption.
  4. After checking all numbers, the assumed largest is the actual largest.
// Find Largest Number — JavaScript
function findLargest(numbers) {
  let largest = numbers[0]; // Step 1: Assume first is largest

  for (let i = 1; i < numbers.length; i++) { // Step 2: Loop
    if (numbers[i] > largest) {              // Step 3: Compare
      largest = numbers[i];                  // Update if bigger
    }
  }
  return largest; // Step 4: Return result
}

console.log(findLargest([3, 7, 1, 9, 4])); // Output: 9

Understanding Time Complexity (Big O)

When we analyze algorithms, we measure how their performance changes as the input size (n) grows. This is called Big O Notation. The algorithm above checks every single element once — so it takes time proportional to the number of elements. We call this O(n) — Linear Time.

Figure: O(n) grows linearly. O(1) is constant — always the same time.

Common Time Complexities

  • O(1) — Constant: Accessing an array by index. Always instant.
  • O(log n) — Logarithmic: Binary Search. Halves the problem each step.
  • O(n) — Linear: Scanning a list once. Like finding the max above.
  • O(n log n) — Linearithmic: Merge Sort & Quick Sort. Very efficient for sorting.
  • O(n²) — Quadratic: Bubble Sort. Nested loops — avoid on large datasets.
  • O(2ⁿ) — Exponential: Brute force solutions. Only works for tiny inputs.
