Bellman-Ford Algorithm

The Bellman-Ford algorithm is another single-source shortest path algorithm. While slower than Dijkstra's algorithm, it is incredibly powerful because it can handle negative weight edges and can detect negative weight cycles.

How it Works (Edge Relaxation)

Bellman-Ford works on the principle of "relaxing" all edges `V - 1` times (where V is the number of vertices). If the shortest path between any two nodes exists without a cycle, it can contain at most `V - 1` edges.

  1. Initialize all distances to `Infinity`, except the start node which is `0`.
  2. Loop `V - 1` times.
  3. In each iteration, go through every single edge in the graph.
  4. If `distance[u] + weight(u, v) < distance[v]`, update `distance[v]`.

Detecting Negative Cycles

A negative weight cycle is a loop of edges where the sum of the weights is negative. If you traverse this cycle infinitely, your distance will keep decreasing to negative infinity. Shortest paths are undefined in graphs with reachable negative cycles.

To detect this: run the relaxation loop one more time (the `V-th` time). If any distance updates during this final pass, a negative cycle exists!

// Bellman-Ford Algorithm (JavaScript)
function bellmanFord(vertices, edges, start) {
  let distances = new Array(vertices).fill(Infinity);
  distances[start] = 0;
  
  // Relax all edges V-1 times
  for (let i = 0; i < vertices - 1; i++) {
    for (let [u, v, weight] of edges) {
      if (distances[u] !== Infinity && distances[u] + weight < distances[v]) {
        distances[v] = distances[u] + weight;
      }
    }
  }
  
  // 1 more time to check for negative cycles
  for (let [u, v, weight] of edges) {
    if (distances[u] !== Infinity && distances[u] + weight < distances[v]) {
      console.log("Graph contains a negative weight cycle!");
      return null;
    }
  }
  
  return distances;
}
