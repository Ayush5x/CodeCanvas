Cycle Detection in Graphs

A cycle occurs in a graph when there is a path from a node back to itself. Detecting cycles is a classic graph problem with different approaches depending on whether the graph is Directed or Undirected.

1. Undirected Graphs

In an undirected graph, an edge implies a connection in both directions. To detect a cycle, we can use DFS or BFS. If we visit a node that has already been visited, and it is not the immediate parent of the current node, we have found a cycle.

// Undirected Graph Cycle Detection (DFS)
function hasCycleUndirected(graph, node, visited, parent) {
  visited.add(node);
  
  for (let neighbor of graph.get(node)) {
    if (!visited.has(neighbor)) {
      if (hasCycleUndirected(graph, neighbor, visited, node)) {
        return true;
      }
    } else if (neighbor !== parent) {
      // We found a visited node that isn't our parent! Cycle!
      return true;
    }
  }
  return false;
}

2. Directed Graphs

In a directed graph, finding a visited node isn't enough, because it might just be a cross edge. We must track nodes that are currently in the recursion stack (i.e., we are actively exploring their descendants). If we encounter a node that is currently in our recursion stack, we have found a back-edge, which means a cycle exists.

// Directed Graph Cycle Detection
function hasCycleDirected(graph, node, visited, recStack) {
  visited.add(node);
  recStack.add(node);
  
  for (let neighbor of graph.get(node)) {
    if (!visited.has(neighbor)) {
      if (hasCycleDirected(graph, neighbor, visited, recStack)) {
        return true;
      }
    } else if (recStack.has(neighbor)) {
      // Node is in the current path stack! Cycle!
      return true;
    }
  }
  
  // Remove from stack before returning
  recStack.delete(node);
  return false;
}
