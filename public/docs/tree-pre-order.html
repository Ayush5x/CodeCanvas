Pre-order Traversal

In Pre-order Traversal, the algorithm visits the Root node first, then recursively visits the Left subtree, and finally the Right subtree.

Traversal Order: Root -> Left -> Right

Because the Root is processed *before* its children, this is called Pre-order traversal. This method is incredibly useful when you need to clone or copy a tree. By processing the root first, you can immediately create a copy of the root node before moving down to clone its branches.

Figure: Pre-order sequence will be: 1, 2, 3, 4, 5

Code Example

Pre-order traversal is usually implemented recursively because it naturally aligns with the Call Stack:

function preOrder(node) {
  if (node === null) return;
  
  // 1. Visit the root
  console.log(node.data);
  
  // 2. Traverse left subtree
  preOrder(node.left);
  
  // 3. Traverse right subtree
  preOrder(node.right);
}
