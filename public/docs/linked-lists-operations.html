Linked Lists Operations

The core operations for a Linked List are Traversal, Insertion, Deletion, and Searching. Understanding the Time Complexity of these operations is key to knowing when to use a Linked List instead of an Array.

1. Traversal (O(n))

Unlike an array, you cannot jump straight to the 5th element. You must start at the head and follow the pointers until you reach the desired node or the end of the list (null).

let current = head;
while (current !== null) {
  console.log(current.data);
  current = current.next;
}

2. Insertion (O(1) to O(n))

Inserting a node is where Linked Lists shine. If you already have a pointer to the location where you want to insert, it takes O(1) time because you only need to update two pointers, without shifting any other elements in memory.

  • Insert at Head: O(1) - Just make the new node point to the current head, and update the head pointer.
  • Insert at Tail: O(1) if you maintain a tail pointer. Otherwise, O(n) to traverse to the end.
  • Insert in Middle: O(n) because you must traverse to the middle first. The actual insertion is O(1).

3. Deletion (O(1) to O(n))

Similar to insertion, deleting a node doesn't require shifting elements. You simply change the next pointer of the previous node to skip the node you want to delete.

// Deleting a node in the middle (assuming we have 'prevNode')
let nodeToDelete = prevNode.next;
prevNode.next = nodeToDelete.next;
// In languages like C++, you must explicitly free(nodeToDelete)

Time Complexity Comparison

Operation Array Linked List
Read / Access O(1) O(n)
Insert/Delete at Beginning O(n) O(1)
Insert/Delete in Middle O(n) O(n) to find, O(1) to execute
