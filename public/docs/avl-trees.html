DSA AVL Trees

An AVL Tree is a self-balancing Binary Search Tree where the height difference between left and right subtrees for any node cannot be more than one.

Why AVL Trees?

A standard Binary Search Tree can become skewed (like a linked list) if elements are inserted in a sorted order. In that worst case, searching takes O(N) time. AVL trees prevent this by auto-balancing themselves during insertions and deletions, guaranteeing O(log N) time for all operations.

Rotations

To keep the tree balanced, AVL trees use rotations when the balance factor goes out of the range [-1, 0, 1]. There are four types of rotations:

  • LL Rotation: Left-Left Rotation. Used when a node is inserted into the left subtree of the left child.
  • RR Rotation: Right-Right Rotation. Used when a node is inserted into the right subtree of the right child.
  • LR Rotation: Left-Right Rotation.
  • RL Rotation: Right-Left Rotation.

Code Structure Idea

class AVLNode {
  constructor(data) {
    this.data = data;
    this.left = null;
    this.right = null;
    this.height = 1; // Used to calculate balance factor
  }
}

// Balance factor = height(left subtree) - height(right subtree)
