DSA Graphs

A Graph is a non-linear data structure consisting of vertices (nodes) and edges that connect these vertices. It is used to represent relationships between objects.

Graph Structure

A vertex, also called a node, is a point or an object in the Graph, and an edge is used to connect two vertices with each other.

Graphs are non-linear because the data structure allows us to have different paths to get from one vertex to another, unlike linear data structures like Arrays or Linked Lists.

They are widely used to model networks such as roads, social media connections, and the internet.

Figure: An undirected graph with 6 vertices.

Graph Representations

  • Adjacency Matrix: A 2D array of size V x V where V is the number of vertices. If there is an edge from vertex i to vertex j, then matrix[i][j] = 1.
  • Adjacency List: An array (or dictionary) of lists. Each key is a vertex, and the list contains all vertices it connects to. This is generally more space-efficient for sparse graphs.

Code Example

Implementation of an Undirected Graph using an Adjacency List in JavaScript:

class Graph {
  constructor() {
    this.adjacencyList = {};
  }
  
  addVertex(vertex) {
    if (!this.adjacencyList[vertex]) {
      this.adjacencyList[vertex] = [];
    }
  }
  
  addEdge(v1, v2) {
    // Add v2 to v1's list
    this.adjacencyList[v1].push(v2);
    // Add v1 to v2's list (because it's undirected)
    this.adjacencyList[v2].push(v1);
  }
  
  removeEdge(v1, v2) {
    this.adjacencyList[v1] = this.adjacencyList[v1].filter(
      v => v !== v2
    );
    this.adjacencyList[v2] = this.adjacencyList[v2].filter(
      v => v !== v1
    );
  }
}

// Usage:
const g = new Graph();
g.addVertex("Dallas");
g.addVertex("Tokyo");
g.addEdge("Dallas", "Tokyo");
