Graphs Implementation

There are two primary ways to represent a graph in computer memory: an Adjacency Matrix and an Adjacency List. Choosing the right one depends heavily on the density of the graph and the operations you need to perform.

1. Adjacency Matrix

An adjacency matrix is a 2D array of size `V x V` where `V` is the number of vertices. If there is an edge connecting vertex `i` to vertex `j`, then `matrix[i][j] = 1` (or the weight of the edge). Otherwise, it is 0.

  • Pros: Checking if an edge exists between two nodes is ultra-fast `O(1)`. Easy to implement.
  • Cons: High memory consumption `O(V^2)`. Very bad for sparse graphs (graphs with few edges) because most of the matrix will be filled with zeros, wasting space.
// Example of an Adjacency Matrix in JS
const graph = [
  [0, 1, 1, 0], // Node 0 is connected to 1 and 2
  [1, 0, 0, 1], // Node 1 is connected to 0 and 3
  [1, 0, 0, 1], // Node 2 is connected to 0 and 3
  [0, 1, 1, 0]  // Node 3 is connected to 1 and 2
];

2. Adjacency List

An adjacency list is an array of lists (or a Map/Dictionary). Each index represents a vertex, and the list at that index contains all the neighboring vertices it is connected to.

  • Pros: Saves massive amounts of memory for sparse graphs. Space complexity is `O(V + E)`. Finding all neighbors of a node is very fast.
  • Cons: Checking if a specific edge exists between node A and B takes `O(V)` in the worst case (you must search the neighbor list).
// Example of an Adjacency List using a Map in JS
const graph = new Map();

graph.set('A', ['B', 'C']);
graph.set('B', ['A', 'D']);
graph.set('C', ['A', 'D']);
graph.set('D', ['B', 'C']);
